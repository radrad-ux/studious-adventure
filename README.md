# صفحة طلب الدفع عند الاستلام — مفتاح صامولة أمان Audi

موقع COD (الدفع عند الاستلام) كامل: صفحة هبوط + سيرفر Node.js يرسل الطلبات تلقائياً لجدول Google Sheets،
مع حماية ضد البوتات والطلبات الوهمية.

---

## 1) كيفاش تشتغل الحماية ضد الطلبات الوهمية / البوتات

| الحماية | كيفاش تخدم |
|---|---|
| **Honeypot field** | فيه حقل مخفي بصرياً (`website`) موجود فالفورم. البشر ما يشوفوه، لكن البوتات اللي تعبي كل الحقول تلقائياً تكتب فيه حاجة → السيرفر يرفض الطلب بصمت. |
| **Time-trap token موقّع** | لما تفتح الصفحة، السيرفر يعطيها "تذكرة" بالوقت موقعة بـ HMAC. إذا الطلب وصل فـ أقل من 4 ثواني (بوت يعبي ويرسل فوراً) → مرفوض. إذا التذكرة قديمة (أكثر من 30 دقيقة) → مرفوض. ما يمكنش تزوّرها لأن موقعة بمفتاح سري ما يبانش للمستخدم. |
| **Captcha رياضي بسيط** | سؤال بسيط (مثلاً 3 + 5) يتولد من السيرفر ويتحقق من الجواب فالسيرفر، بلا حاجة لخدمة خارجية (Google reCAPTCHA ممكن تزيدها بعدين إذا حبيت). |
| **Rate limiting** | أي IP وحد ما يقدرش يرسل أكثر من 5 طلبات كل 15 دقيقة. يوقف البوتات اللي تحاول تبعث مئات الطلبات. |
| **فحص رقم الهاتف الجزائري** | يقبل غير أرقام تبدأ بـ 05 / 06 / 07 ومكوّنة من 10 أرقام. أي رقم خاطئ يترفض قبل ما يوصل للجدول. |
| **كشف التكرار (Duplicate detection)** | إذا نفس رقم الهاتف طلب نفس المنتج فآخر 24 ساعة، الطلب ما يترفضش، لكن يتسجل فالجدول بعلامة "تكرار محتمل" فعمود Flags باش تنتبه لها قبل التأكيد. |
| **تسجيل IP + User-Agent** | كل طلب يتسجل مع IP والمتصفح المستعمل، يساعدك تكتشف إذا نفس الشخص يبعث بأسماء مختلفة. |
| **التأكيد اليدوي بالهاتف** | كل طلب يدخل بحالة "بانتظار التأكيد" — أنت لي تأكدها يدوياً فالجدول بعد المكالمة (Confirmed / Cancelled). هذا أهم خط دفاع ضد الطلبات الوهمية. |

> ملاحظة: ما كاين حماية 100%. الهدف هو رفع التكلفة على البوتات والطلبات العشوائية باش يبقى عندك غير الطلبات اللي تستاهل مكالمة تأكيد.

## 2) إعداد Google Sheets خطوة بخطوة

### أ) أنشئ الجدول (Spreadsheet)
1. روح لـ [sheets.google.com](https://sheets.google.com) وأنشئ Spreadsheet جديد، سميه مثلاً **"طلبات AntiVolDZ"**.
2. فالصف الأول (Row 1) دير هذي العناوين بالضبط، بنفس الترتيب:
   ```
   Timestamp | Name | Phone | Wilaya | Address | Product | Qty | UnitPrice | Total | Status | Flags | IP | UserAgent
   ```
3. سمي الـ Sheet (التبويب فالأسفل) `Orders` (هو الاسم المستعمل فالكود، أو بدّله فـ `.env`).
4. خذ الـ **Sheet ID** من الرابط:
   ```
   https://docs.google.com/spreadsheets/d/RABT_HNA_HOWA_SHEET_ID/edit
   ```

### ب) أنشئ Service Account (حساب خدمة)
هذا حساب خاص بالبرنامج (ماشي حسابك الشخصي) باش يقدر يكتب فالجدول تلقائياً.

1. روح لـ [console.cloud.google.com](https://console.cloud.google.com)، سجل دخول بنفس حساب Google متاعك.
2. فوق فالشريط، دوس على اختيار المشروع → **New Project** → سميه مثلاً `antivol-dz` → Create.
3. من القائمة الجانبية: **APIs & Services → Library**. دور على **Google Sheets API** ودوس **Enable**.
4. من القائمة الجانبية: **APIs & Services → Credentials**.
5. دوس **Create Credentials → Service Account**.
   - الاسم: `cod-sheets-bot` (أي اسم).
   - دوس **Create and Continue**، بعدها **Done** (بلا أدوار خاصة، ماشي ضروري).
6. فقائمة Service Accounts، دوس على الحساب اللي خلقتيه.
7. روح لتبويب **Keys** → **Add Key → Create new key** → اختار **JSON** → Create.
   - باش يتحمّل عندك ملف `.json` — هذا فيه السر، ما تشاركوش حتى لحد.
8. حل الملف بأي محرر نصوص. غادي تلقى فيه:
   ```json
   {
     "client_email": "cod-sheets-bot@antivol-dz.iam.gserviceaccount.com",
     "private_key": "-----BEGIN PRIVATE KEY-----\nMIIEvQI...\n-----END PRIVATE KEY-----\n",
     ...
   }
   ```
   - `client_email` → هذا غادي يكون `GOOGLE_SERVICE_ACCOUNT_EMAIL`
   - `private_key` → هذا غادي يكون `GOOGLE_PRIVATE_KEY` (خليه بالضبط كيما هو، بـ `\n`)

### ج) شارك الجدول مع الـ Service Account
1. رجع لـ Google Sheet اللي خلقتيه.
2. دوس **Share** (مشاركة) فالزاوية العليا.
3. لصق الـ `client_email` (مثال: `cod-sheets-bot@antivol-dz.iam.gserviceaccount.com`).
4. خليه صلاحية **Editor**، إلغي "Notify people"، دوس **Share**.

بهذا، البرنامج عندو الحق يكتب فالجدول.

## 3) النشر (Deploy) على Render

1. حط هذا المجلد فـ GitHub repo (خاص ولا عام، الاختيار ليك).
2. روح لـ [render.com](https://render.com) → **New → Web Service**.
3. اربط الـ repo متاعك.
4. الإعدادات:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Instance Type:** Free يكفي للبداية (يدوّر فهدوء إذا ما كاين حركة، يحتاج ~30 ثانية باش يصحى — إذا حبيت أسرع، اختار Starter).
5. فـ **Environment** زيد المتغيرات التالية (انسخها من `.env.example`):
   - `GOOGLE_SHEET_ID`
   - `GOOGLE_SHEET_TAB` = `Orders`
   - `GOOGLE_SERVICE_ACCOUNT_EMAIL`
   - `GOOGLE_PRIVATE_KEY` (الصقها بالكامل بـ `\n` كيما هي فالملف JSON، بين علامتي تنصيص)
   - `CHALLENGE_SECRET` (ولّد وحدة بـ: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`)
6. دوس **Create Web Service**. بعد البناء، تلقى الموقع حي على رابط زوج `https://your-app.onrender.com`.

## 4) التجربة محلياً قبل النشر (اختياري)

```bash
npm install

# عمر القيم فـ .env
npm start
# افتح http://localhost:3000
```

## 5) تخصيص الموقع

- **السعر**: بدّل `UNIT_PRICE` فآخر `public/index.html` (السطر فيه `var UNIT_PRICE = 2900;`) وبدّل أيضاً السعر المعروض فالـ `.price-tag`.
- **اسم المنتج / المواصفات**: بدّل النصوص مباشرة فـ `public/index.html`.
- **رقم الهاتف للتواصل المباشر**: زيد رابط واتساب أو رقم فالـ footer إذا حبيت.
- **عدد الطلبات المسموحة كل 15 دقيقة لكل IP**: بدّل `max: 5` فـ `server.js` (دالة `orderLimiter`).
- **مدة كشف التكرار**: بدّل `DEDUPE_WINDOW_HOURS` فـ `.env`.

---

## 6) كيفاش تأكد الطلبات يومياً

1. افتح Google Sheet.
2. أي طلب جديد يبان بحالة **بانتظار التأكيد**.
3. اتصل بالزبون، وبعد المكالمة بدّل عمود **Status** يدوياً لـ:
   - `مؤكد` إذا الزبون أكد الطلب.
   - `ملغى` إذا ما ردش أو رفض.
4. انتبه للطلبات اللي عندها علامة فعمود **Flags** ("تكرار محتمل") — تأكد منها بعناية أكثر قبل الشحن.
